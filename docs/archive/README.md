# Архив документации

Эта папка содержит устаревшие документы, которые были перемещены из основной документации.

## Содержимое архива

### Устаревшие планы
- `PLAN_OFFLINE_MODE.md` - первоначальный план реализации офлайн-режима (информация вынесена в отдельные документы)
- `PLAN_OFFLINE_MODE_IMPROVEMENTS.md` - предложения по улучшению плана (предложения реализованы)
- `OFFLINE_SYNC_PLAN.md` - детальный план синхронизации (информация вынесена в актуальные документы)

### Заметки
- `OFFLINE_SYNC_NOTES.md` - заметки по синхронизации (информация вынесена в `SYNC_ALGORITHM.md`)
- `OFFLINE_CACHE_NOTES.md` - заметки по кэшу (информация вынесена в `OFFLINE_FIRST_ARCHITECTURE.md`)
- `OFFLINE_DAY_LOGIC_NOTES.md` - заметки по логике дня (информация вынесена в `OFFLINE_REQUIREMENTS.md`)
- `NOTES.md` - общие заметки (PowerShell, pytest и т.д.)
- `sync_offline_notes.md` - заметки по синхронизации из knowledge_base

## Когда использовать архив

Архивные документы могут быть полезны для:
- Понимания истории развития проекта
- Анализа принятых решений
- Ссылок на старые подходы

**Важно:** Архивные документы не должны использоваться как источник актуальной информации. Для актуальной информации используйте документы в основной папке `docs/`.

## Актуальные документы

Актуальная информация находится в следующих документах:
- `OFFLINE_FIRST_ARCHITECTURE.md` - архитектура офлайн-режима
- `OFFLINE_REQUIREMENTS.md` - требования к офлайн-режиму
- `OFFLINE_CLIENT_DESIGN.md` - архитектура клиента
- `SYNC_ALGORITHM.md` - алгоритм синхронизации
- `SYNC_POLICY.md` - политика синхронизации
